JSON vs XML

Kada je oblast primene u pitanju, JSON, kao i XML standard se najčešće mogu prinaći u komunikaciji između web servera i klijenta. Konkretno, koriste se za primanje podataka poslatih od strane web servera. U primerima koji slede možete uporediti JSON sintaksu sa sintaksom XML standarda, kojom je kreiran objekat employees, koji u sebi sadrži niz od nekoliko radnika.

JSON primer

  • {"employees":[
  • { "firstName":"John", "lastName":"Doe" },
  • { "firstName":"Anna", "lastName":"Smith" },
  • { "firstName":"Peter", "lastName":"Jones" }
  • ]}

XML primer

  • <employees>
  •  <employee>
  •   <firstName>John</firstName>
  •   <lastName>Doe</lastName>
  •  </employee>
  •  <employee>
  •   <firstName>Anna</firstName>
  •   <lastName>Smith</lastName>
  •  </employee>
  •  <employee>
  •   <firstName>Peter</firstName>
  •   <lastName>Jones</lastName>
  •  </employee>
  • </employees>

Razlike

JSON ima veoma jednostavnu strukturu, što nije odlika XML standarda, te mu nisu potrebni završni tagovi. Iz prethodne osobine proizilaze i sledeće dve. Ukoliko su u pitanju velike količine podataka koje je potrebno preneti JSON se pokazao veoma dobro, jer je sam JSON kod kraći te će zahtevati manje memorije, a prenos će biti brži. Zbog svoje jednostavne strukture i čitljivosti koda, JSOM je brži i lakši za pisanje.

Kao jednu od bitnijih razlika, naveli bismo i to da JSON u svojoj strukturi ima ugrađenu podršku za nizove objekata, što znatno olakšava rad sa grupama podataka, dok je sa druge strane XML standard, čija struktura nema podršku za nizove, te je prilično teško odrediti kada je u pitanju niz podataka, a kada su podaci samo poređani jedan do drugog.

Sličnosti

Za JSON, kao i za XML, se može reći da su samo-opisujući, tačnije veoma su čitljivi za korisnika. Te korisnik sa lakoćom, na prvi pogled, može razumeti šta je svrha datog objekta. Oba standarda su hijerarhijski uređena, i svaki objekat se može sastojati od svojih podobjekata. Podacima predstavljenim u bilo kojoj od ove dve forme moguće je pristupiti korišćenjem XMLHttpRequest-a.

Mnogi programski jezici, tačnije velika većina, podržavaju rad sa oba standarda, što nam daje do znanja da su oba jezika veoma rasprostranjena, ali da svaki ima kako svoje prednosti tako i svoje mane.

Gde se JSON pokazao bolje?

XML je, zbog svoje složene strukture, znatno komplikovaniji za raščlanjivanje i izvlačenje podataka, u odnosu na JSON. Kada raščlanite JSON zapis, dobijate JavaScript objekat koji je spreman za korišćenje, što se ne može reći i za XML zapise.

XML je, zbog svoje složene strukture, znatno komplikovaniji za raščlanjivanje i izvlačenje podataka, u odnosu na JSON. Kada raščlanite JSON zapis, dobijate JavaScript objekat koji je spreman za korišćenje, što se ne može reći i za XML zapise.

Kod korišćenja XML zapisa prvo je potrebno dostaviti XML dokument, zatim koristiti XML DOM da bi se prošlo kroz čitav objekat, i na kraju je potrebno izvući i smestiti vrednosti u varijable. Kod JSON zapisa je ovaj proces malo drugačiji, jer nije potrebno dostaviti čitav dokument u kome se nalazi zapis, već je dovoljno poslati samo zapis u vidu stringa, te zatim taj zapis raščlaniti na ostale objekte.
